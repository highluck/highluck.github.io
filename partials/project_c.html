Write by highluck

 이번엔 전포스팅에서 이야기한 보내는 서버를 개발해야 한다.


- 자만심이 부른 바보짓거리 -

설계를 미리 해놓았기 때문에 바로 프로젝트를 만들고 패키지를 구성하고 개발을 시작!!...하자마자 난관에 봉착했다. 처음 접했을때 문법이 생각보다 어려웠다. 처음으로 겪은 문제는 다음 예시같은 문제 였다.


val a = 3 //선언
a = 6 //다시 값 정의

헉 안된다... 뭐지 뭐지???

그리고 또...


def Add(a:int, b:Int) : Int = {
    return a + b
}

헉 안된다... 뭐지 뭐지???


 그렇다.. 나는 자만했다. 자만하고 공부를 안하고 스칼라를 시작했다.. 사실 되게 간단한 문제였다. 스칼라에는 기본적으로 val과 var 두가지 타입이 있는데 val 은 Immutable 즉 getter만 있고 setter는 없는 타입이다. setter와 getter를 모두 사용하려면 var를 사용해야했다.

 두번째 문제는 더 간단했다. 스칼라는 기본적으로 마지막 값을 return 해준다. 정말 내가 생각해도 황당한 바보짓거리 였다. 자만심이 부른 비극... 하지만 이대로 포기하지 않는다. 잠시 멈추고 공부를 한 후 다시 개발을 시작했다.


- 이것은 Java인가.. Scala인가.. -


 틈틈히 책을 정독하고 다시 Scala로 개발을 시작했다. 그리고 험난한 여정끝에 개발은 무사히 마쳤지만 개발을 모두 마친후 나의 코드를 보니... 이게 자바인지 스칼라인지.. 스칼라를 참 자바처럼 코딩해버렸다. OTL.. 물론 이전 게임회사에서 C#을해서 Lambda를 좀 사용해서 Lambda를 사용하는 것에 거부감은 없었지만 나스스로가 함수형이란 개념을 잘몰랐던 것 같다. 그러다보니 Java컬렉션만 잔득사용한 Javcala(잡칼라)가 되버렸다.

근데 스칼라! 매력적이고 재밌었다. 스칼라는 집에서 혼자 공부를 더 해야겠다. 당분간은 Java를 열심히 해야겠다.
