Redis MySQL Batch Scrapping

 사실 레디스를 주로 사용하는 이유는 캐싱이다


많은 량의 컨텐츠를 한화면에 보여주는데 어떻게 해야 속도가 올라갈까?

그 해답은 캐싱일 것이다. 병렬처리를 해도 매번 API를 호출하고 DB커넥션을 맺는것은 엄청난 비용이다.
그래서 우리는 주기적으로 배치에서 캐싱을 한다. 레디스는 InMemory엔진이기 때문에 속도가 빠르다 Rom과 Ram의 속도 차이라고 생각하면 됨


주로 배치에서 주기적으로 캐싱을하고 캐싱이 안되었을때는 DB에 접근해서 캐시를 하고 return을 해주는 식으로 캐싱이 안된 데이터를 캐싱 해준다. Hset같은 redis 컬렉션을 안쓰는 이유는 expireTime 때문인데 hset은 하나의 키에 n개의 hash키가 저장되기 때문에 exprieTime관리가 안된다. 그래서 캐싱을 할때는 그냥 set을 쓰는게 맞는것 같다.


저장소 용도로 쓸대는 무조껀 Hset을 권장한다.


레디스는 인메모리 엔진이기 때문에 하나의 key가 먹는 리소스가 커서 메모리릭을 발생할 수 있다. 그래서 Hset을쓰면 하나의 키에 N개의 데이터를 가질 수 있기때문에 저장소용도를 쓸때는 무조껀 Hset이나 레디스 컬렉션을 사용하길 바란다.
